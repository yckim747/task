# 업무 관리 웹 서비스 개발 명세서

## 프로젝트 개요

업무 항목을 등록·조회·정렬·출력할 수 있는 반응형 단일 페이지 웹 서비스.  
별도 백엔드 없이 브라우저 로컬스토리지(LocalStorage)를 데이터 저장소로 사용한다.  
기술 스택은 **HTML + CSS + Vanilla JS** 단일 파일(`index.html`)로 구현한다.

---

## 기술 요건

| 항목 | 내용 |
|------|------|
| 구현 방식 | 단일 `index.html` (HTML + CSS + JS 인라인) |
| 데이터 저장 | `localStorage` |
| 반응형 | 데스크톱 / 태블릿(768px) / 모바일(480px) 브레이크포인트 |
| 외부 라이브러리 | SheetJS(xlsx) — CDN 로드, 그 외 의존성 없음 |
| 브라우저 지원 | Chrome / Edge / Safari 최신 버전 |

---

## 데이터 구조

업무 1건을 아래 필드로 구성한다.

```js
{
  id: "uuid-v4",           // 자동 생성, 불변
  title: "",               // 타이틀 (필수)
  detail: "",              // 상세 (멀티라인 텍스트)
  department: "",          // 유관부서
  priority: "",            // 우선순위: "높음" | "보통" | "낮음"
  status: "",              // 진행상태: "대기" | "진행중" | "검토" | "완료" | "보류"
  startDate: "",           // 착수예정일 (YYYY-MM-DD)
  endDate: ""              // 완료예정일 (YYYY-MM-DD)
}
```

## 화면 구성

### 1. 헤더 영역

- 서비스 타이틀 표시
- **[+ 업무 추가]** 버튼 — 클릭 시 등록 모달 오픈
- **[엑셀 출력]** 버튼 — 현재 정렬/필터 상태 그대로 xlsx 다운로드

### 2. 필터/검색 바

- 텍스트 검색 입력창: 타이틀·상세·유관부서를 대상으로 실시간 필터링
- 진행상태 드롭다운 필터: 전체 / 대기 / 진행중 / 검토 / 완료 / 보류
- 우선순위 드롭다운 필터: 전체 / 높음 / 보통 / 낮음

### 3. 업무 목록 테이블

컬럼 순서 및 정렬 가능 여부:

| # | 컬럼명 | 필드 | 정렬 |
|---|--------|------|------|
| 1 | 타이틀 | `title` | ✅ |
| 2 | 유관부서 | `department` | ✅ |
| 3 | 우선순위 | `priority` | ✅ |
| 4 | 진행상태 | `status` | ✅ |
| 5 | 착수예정일 | `startDate` | ✅ |
| 6 | 완료예정일 | `endDate` | ✅ |
| 7 | 작업 | — | ❌ |

- 헤더 셀 클릭 시 해당 컬럼 기준 **오름차순 → 내림차순 → 정렬 해제** 순으로 토글
- 현재 정렬 기준 컬럼에 ▲ / ▼ 아이콘 표시
- 행 클릭(작업 버튼 제외) 시 상세 보기/수정 모달 오픈
- 우선순위·진행상태는 색상 배지(badge)로 표시

**우선순위 배지 색상**

| 값 | 색상 |
|----|------|
| 높음 | 빨강 계열 |
| 보통 | 주황/노랑 계열 |
| 낮음 | 초록 계열 |

**진행상태 배지 색상**

| 값 | 색상 |
|----|------|
| 대기 | 회색 |
| 진행중 | 파랑 |
| 검토 | 보라 |
| 완료 | 초록 |
| 보류 | 주황 |

- 목록이 비어 있을 때 "등록된 업무가 없습니다" 빈 상태 메시지 표시
- 하단에 전체 건수 표시 (예: `총 12건`)

### 4. 업무 등록/수정 모달

- 오버레이 배경 + 중앙 정렬 카드형 모달
- 모달 외부 클릭 또는 [✕] 버튼 클릭 시 닫힘
- 입력 필드

| 필드 | UI 컴포넌트 | 필수 여부 |
|------|-------------|-----------|
| 타이틀 | `<input type="text">` | ✅ 필수 |
| 상세 | `<textarea rows="4">` | 선택 |
| 유관부서 | `<input type="text">` | 선택 |
| 우선순위 | `<select>` (높음/보통/낮음) | ✅ 필수 |
| 진행상태 | `<select>` (대기/진행중/검토/완료/보류) | ✅ 필수 |
| 착수예정일 | `<input type="date">` | 선택 |
| 완료예정일 | `<input type="date">` | 선택 |

- **[저장]** 버튼: 유효성 검사 통과 시 localStorage에 저장 후 목록 갱신, 모달 닫기
- **[취소]** 버튼: 변경사항 없이 모달 닫기
- 수정 모드: 모달 타이틀이 "업무 수정"으로 변경되고 기존 값이 채워진 상태로 열림
- 행의 **[삭제]** 버튼 클릭 시 확인 다이얼로그 후 삭제

## 정렬 로직

```
1. 정렬 컬럼이 없으면 등록 순서(id 기준) 유지
2. 문자열 필드: localeCompare() 사용 (한글 포함 처리)
3. 날짜 필드: 날짜 없는 항목은 오름차순 시 맨 뒤, 내림차순 시 맨 앞으로 배치
4. 우선순위 정렬 가중치: 높음=1, 보통=2, 낮음=3
5. 진행상태 정렬 가중치: 진행중=1, 대기=2, 검토=3, 보류=4, 완료=5
```

## 엑셀 출력 명세

- 라이브러리: **SheetJS (xlsx)** CDN  
  `https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js`
- 출력 파일명: `업무목록_YYYYMMDD.xlsx`
- 출력 대상: 현재 화면에 표시 중인 데이터(검색 필터 + 정렬 반영)
- 시트명: `업무목록`
- 헤더 행(1행) 굵게, 배경색 적용 (SheetJS 스타일 지원 범위 내)

| 열 | 헤더 |
|----|------|
| A | 타이틀 |
| B | 상세 |
| C | 유관부서 |
| D | 우선순위 |
| E | 진행상태 |
| F | 착수예정일 |
| G | 완료예정일 |

- 날짜는 `YYYY-MM-DD` 문자열로 출력
- 빈 필드는 빈 문자열로 출력

## 반응형 브레이크포인트

| 구간 | 처리 방식 |
|------|-----------|
| `> 768px` | 테이블 전체 컬럼 표시 |
| `768px 이하` | 상세·착수예정일 컬럼 숨김, 가로 스크롤 허용 |
| `480px 이하` | 모달 전체 화면 전환, 버튼 전체 너비 |

## 디자인 방향

- 톤: **실용적 미니멀** — 업무 도구답게 깔끔하고 밀도 있게
- 배경: 연한 중성 톤 (off-white 또는 light gray)
- 포인트 컬러: 딥 인디고 또는 슬레이트 블루 계열 1색
- 폰트: Google Fonts에서 한글 지원 폰트 사용 (예: `Noto Sans KR`)
- 테이블 행: hover 시 배경색 미세 변화
- 버튼: 기본/호버/액티브 상태 전환 트랜지션 (0.15s ease)
- 모달: 열릴 때 fade-in + slight scale-up 애니메이션
